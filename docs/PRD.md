# PRD - 逢甲大學校園美食資訊平台

## 1. 專案概述
### 背景與動機
逢甲大學周邊美食林立，但每逢用餐時段經常面臨排隊候位久、各家餐廳資訊散亂無法即時取得等問題。為了解決這些痛點，本系統旨在為逢甲師生打造一個整合性的校園美食資訊平台，透過數位化管理提升師生的用餐體驗，同時也幫助周邊店家提升經營效率。

### 目標用戶
- 逢甲大學學生
- 逢甲大學老師
- 校園周邊合作店家管理者

### 核心價值主張
提供一站式的校園美食探索與訂餐服務，減少排隊等待時間，並透過真實評價建立可靠的社群美食指南。

## 2. 功能需求

### 1. 店家資訊探索
- **使用者故事**：作為學生或老師，我希望能夠快速分類瀏覽或搜尋校園周邊美食店家資訊（如營業時間、距離、餐點類型），以便找到符合當下口味與預算的餐廳。

### 2. 數位菜單與即時價格
- **使用者故事**：作為學生或老師，我希望能夠查看各店家的最新數位菜單與即時價格，以便在前往用餐或點餐前先做好決策。

### 3. 線上預約訂餐系統
- **使用者故事**：作為學生或老師，我希望能夠透過系統提前線上點餐並預約取餐時間，以便節省現場排隊與候位的寶貴時間。

### 4. 真實評價與社群互動
- **使用者故事**：作為學生或老師，我希望能夠查看其他師生的真實用餐評價，並在用餐後分享自己的心得，以便避雷或推薦隱藏版美食給同學。

### 5. 後台管理儀表板
- **使用者故事**：作為店家管理者，我希望能夠有一個直觀的後台管理介面，以便隨時更新菜單資訊、管理線上訂單與查看顧客評價。

## 3. 非功能需求
- **技術限制**：後端採用 Python Flask 框架，前端視圖使用 Jinja2 模板引擎渲染，資料庫使用輕量級的 SQLite。
- **效能考量**：系統需能負荷中午及傍晚用餐尖峰時段的並行存取，確保頁面載入速度與點餐流程順暢。
- **安全考量**：使用者的密碼需進行雜湊加密處理；對於使用者輸入的評價需進行 XSS 防護處理。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須擁有)
- 店家資訊展示與搜尋（名稱、分類）
- 數位菜單瀏覽與價格顯示

### Should Have (應該擁有)
- 基礎的線上預約訂餐與購物車功能
- 店家簡易版後台管理（菜單上下架、訂單狀態更新）

### Nice to Have (可以擁有)
- 使用者註冊與登入功能
- 真實評價與評分系統
- 點數或優惠券回饋系統
